# Telegram-бот для работы с локальной моделью Qwen

Полноценный Telegram-бот на **aiogram 3.x**, который работает как посредник между пользователем и локальной языковой моделью **Qwen3.5 Q3_K** через **llama-server**.

## Особенности

- 🚀 **Стриминг ответов** — текст приходит по мере генерации токенов (Server-Sent Events)
- 🔒 **Безопасное хранение токена** — через файл `.env` с использованием `python-dotenv`
- 💾 **База данных SQLite** — учёт пользователей с помощью `aiosqlite`
- 📊 **Админ-панель** — просмотр всех зарегистрированных пользователей
- ⌨️ **Кнопки команд** — удобный доступ к функциям бота
- 📝 **Логирование** — все события записываются в `bot.log`
- 🛡️ **Обработка ошибок** — таймауты, отключения сервера, graceful shutdown

## Установка

### 1. Требования

- **Python 3.9+** (проверить: `python --version`)
- **llama-server** (запущен на `http://127.0.0.1:8080`)

### 2. Автоматическая установка (Windows)

Запустите скрипт установки:

```batch
install.bat
```

Скрипт автоматически:
- Создаст виртуальное окружение `venv`
- Установит все зависимости из `requirements.txt`
- Создаст файл `.env` из шаблона

### 3. Ручная установка

```bash
# Создание виртуального окружения
python -m venv venv

# Активация (Windows)
venv\Scripts\activate

# Активация (Linux/Mac)
source venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt

# Создание файла конфигурации
copy .env.example .env      # Windows
cp .env.example .env        # Linux/Mac
```

## Настройка

### 1. Получение токена бота

1. Откройте Telegram и найдите бота **@BotFather**
2. Отправьте команду `/newbot`
3. Придумайте имя и username для бота (должен заканчиваться на `bot`)
4. BotFather выдаст токен вида `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`

### 2. Узнавание своего Telegram ID

1. Найдите бота **@userinfobot**
2. Отправьте любое сообщение
3. Бот покажет ваш ID (число вида `123456789`)

### 3. Заполнение .env

Откройте файл `.env` и заполните параметры:

```env
# Токен от @BotFather
BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz

# URL llama-server (обычно не требует изменений)
LLAMA_SERVER_URL=http://127.0.0.1:8080/v1/chat/completions

# Ваш Telegram ID (для доступа к /users)
OWNER_ID=123456789

# Обрабатывать ли все сообщения как запросы к модели
DEFAULT_MODE=true

# Размер чанка для стриминга (символов)
STREAM_CHUNK_SIZE=50
```

## Запуск llama-server

### Установка llama.cpp

```bash
# Клонирование репозитория
git clone https://github.com/ggerganov/llama.cpp.git
cd llama.cpp

# Сборка (Windows)
cmake -B build -S .
cmake --build build --config Release

# Сборка (Linux/Mac)
make -j
```

### Запуск сервера с моделью Qwen

```bash
# Windows
build\bin\Release\llama-server.exe -m models\qwen-2.5-7b-q3_k_m.gguf -c 4096 --host 127.0.0.1 --port 8080

# Linux/Mac
./server -m models/qwen-2.5-7b-q3_k_m.gguf -c 4096 --host 127.0.0.1 --port 8080
```

**Параметры:**
- `-m` — путь к модели в формате GGUF
- `-c` — контекст (количество токенов)
- `--host` — адрес сервера
- `--port` — порт (должен совпадать с `.env`)

## Команды бота

| Команда | Описание | Доступ |
|---------|----------|--------|
| `/start` | Запустить бота, приветствие | Все |
| `/help` | Справочная информация | Все |
| `/ask <текст>` | Задать вопрос модели | Все |
| `/commands` | Показать кнопки команд | Все |
| `/users` | Таблица всех пользователей | Владелец |

### Кнопки

- **📋 Команды** — показать список команд
- **❓ Помощь** — вызвать справку

## Обработка ошибок

Бот обрабатывает следующие ситуации:

| Ошибка | Реакция |
|--------|---------|
| Таймаут соединения | Сообщение пользователю с просьбой повторить |
| Отключение сервера | Логирование и уведомление |
| Пустой ответ | Предложение попробовать ещё раз |
| KeyboardInterrupt | Graceful shutdown с закрытием соединений |
| 429 (Rate Limit) | Ограничение на `edit_text` не чаще 1.5 сек |

## Troubleshooting

### Бот не отвечает на сообщения

1. Проверьте, что `DEFAULT_MODE=true` в `.env`
2. Убедитесь, что бот запущен (`python main.py`)
3. Проверьте логи в `bot.log`

### Ошибка "Таймаут: сервер не ответил вовремя"

1. Убедитесь, что `llama-server` запущен
2. Проверьте URL в `.env` (должен быть `http://127.0.0.1:8080/v1/chat/completions`)
3. Откройте в браузере `http://127.0.0.1:8080` — должна быть страница сервера

### Ошибка "BOT_TOKEN не найден"

1. Скопируйте `.env.example` в `.env`
2. Заполните `BOT_TOKEN` реальным токеном от @BotFather

### Ошибка "OWNER_ID не настроен"

1. Узнайте свой ID через @userinfobot
2. Заполните `OWNER_ID` в `.env`
